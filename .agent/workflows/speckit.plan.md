# /speckit.plan

将已批准的 spec 转化为可执行的实施计划。

## 前置条件

- 对应的 spec 状态必须为 `✅ 已批准`
- 如果 spec 仍为草稿或审查中状态，拒绝执行并提示用户先完成审批

## 步骤

1. **读取 Spec**
   读取用户指定的 spec 文件（或由用户选择）。

2. **读取项目上下文**
   读取 `docs/project-context/current-context.md` 了解当前架构和约束。

3. **读取宪法**
   读取 `docs/specs/CONSTITUTION.md` 确保计划符合所有原则。

4. **生成实施计划**
   创建包含以下内容的计划文件 `docs/specs/PLAN-{SPEC-NNN}.md`:
   
   ### 计划结构
   ```
   # 实施计划: {Spec 标题}
   
   ## 影响分析
   - 受影响的文件列表
   - 需要新建的文件列表
   - 需要修改的文件列表
   
   ## 实施阶段
   - 阶段 1: {描述}
     - 步骤 1.1
     - 步骤 1.2
   - 阶段 2: {描述}
     - ...
   
   ## 依赖项
   - 新的 npm/pip/etc 依赖
   - 需要的工具或服务
   
   ## 测试计划
   - 单元测试清单
   - 集成测试清单
   - E2E 测试清单
   
   ## 回滚策略
   - 如何回滚每个阶段的变更
   
   ## 风险与缓解
   - 已识别的风险及应对措施
   ```

5. **标记 Spec 状态**
   将 spec 状态更新为 `🚧 实施中`。

6. **输出摘要 (中文)**
   向用户呈现实施计划的中文摘要。

> [!NOTE]
> 计划创建后即可开始使用 `/speckit.tasks` 分解任务。
