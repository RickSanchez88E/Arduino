# /handoff

在结束当前工作会话前创建完整的交接笔记。

## 步骤

1. **收集会话信息**
   回顾当前会话中:
   - 完成了哪些工作
   - 修改了哪些文件
   - 遇到了哪些问题
   - 还有哪些工作未完成

2. **创建交接笔记**
   基于 `docs/handoffs/HANDOFF-TEMPLATE.md` 在 `docs/handoffs/` 中创建:
   ```
   HANDOFF-{YYYY-MM-DD}-{HH-MM}-{简短描述}.md
   ```

3. **填写交接内容**
   必须填写以下所有字段:
   - ✅ Agent 标识
   - ✅ 日期和会话 ID
   - ✅ 完成的工作列表
   - ✅ 未完成的工作列表
   - ✅ 文件变更清单
   - ✅ 已知问题
   - ✅ 上下文和注意事项
   - ✅ 建议的下一步

4. **释放文件锁**
   检查并释放 `docs/agent-board/agent-registry.md` 中当前 agent 持有的所有文件锁。

5. **更新任务状态**
   如果有关联的任务，更新任务状态:
   - 完成的任务标记为 `✅ 完成`
   - 未完成的任务保留为 `🔄 进行中` 或退回 `📋 待认领`

6. **输出摘要 (中文)**
   向用户提供中文摘要:
   - 本次会话完成了什么
   - 下一位 agent 应该做什么
   - 需要注意的风险

> [!IMPORTANT]
> 每个 agent 会话结束前**必须**执行此工作流。这是强制要求。
