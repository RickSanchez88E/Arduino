# /openspec.clarify

可选步骤。当提案不够清晰时，通过对话补充上下文。

## 何时需要

- 提案涉及多个模块但影响不确定
- 需要从已有代码中提取约束
- 用户想先讨论再动手

## 步骤

1. **读取提案**
   读取 `docs/openspec/OS-{NNN}-{slug}/proposal.md`。

2. **提出澄清问题**
   基于提案内容和项目上下文，提出 2-5 个精准问题:
   - 边界确认: 「这个改动是否也要覆盖 X 场景？」
   - 约束确认: 「当前 Y 模块有 Z 约束，是否需要调整？」
   - 取舍确认: 「优先性能还是代码简洁？」

3. **更新提案**
   将澄清结果追加到 `proposal.md` 的「补充说明」部分。
   更新状态为 `🔍 已澄清`。

4. **输出摘要 (中文)**

> [!NOTE]
> 这个步骤是**可选的**。如果提案足够清晰，可以直接跳到 `/openspec.design`。
