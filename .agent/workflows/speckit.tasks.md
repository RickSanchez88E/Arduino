# /speckit.tasks

将实施计划分解为独立的、可并行执行的任务单元。

## 前置条件

- 对应的 `PLAN-{SPEC-NNN}.md` 必须存在
- spec 状态必须为 `🚧 实施中`

## 步骤

1. **读取实施计划**
   读取 `docs/specs/PLAN-{SPEC-NNN}.md`。

2. **任务分解**
   将计划中的每个阶段和步骤分解为任务单元，每个任务满足以下要求:
   - 可独立执行（最小化跨任务依赖）
   - 范围明确（影响的文件有限且确定）
   - 可验证（有明确的完成标准）
   - 时间可控（预估不超过一个 agent 会话）

3. **创建任务文件**
   在 `docs/specs/tasks/` 目录下创建任务文件:
   ```
   TASK-{SPEC-NNN}-{NN}.md
   ```
   
   每个任务文件包含:
   ```markdown
   # 任务: {标题}
   
   > Spec: SPEC-{NNN}
   > 状态: 📋 待认领 | 🔄 进行中 | ✅ 完成 | ❌ 阻塞
   > 优先级: P0/P1/P2
   > 预估: {时间}
   
   ## 描述
   {具体要做什么}
   
   ## 受影响文件
   - {文件列表}
   
   ## 依赖
   - {依赖的其他任务}
   
   ## 验收标准
   - [ ] {标准 1}
   - [ ] {标准 2}
   
   ## 注意事项
   - {来自宪法的约束}
   ```

4. **生成任务看板摘要**
   更新 `docs/agent-board/agent-registry.md` 中的任务概览。

5. **输出摘要 (中文)**
   向用户呈现任务分解的中文摘要，包括:
   - 总共生成多少个任务
   - 任务依赖关系图
   - 可并行执行的任务组
   - 推荐的执行顺序

> [!TIP]
> 没有依赖关系的任务可以分配给不同的 agent 并行执行。
