# /speckit.specify

为新功能或变更创建规范文件。

## 步骤

1. **收集信息**
   向用户收集以下信息:
   - 功能名称和描述
   - 目标和动机
   - 范围（包含和不包含）
   - 预期行为

2. **生成 Spec 编号**
   扫描 `docs/specs/` 目录，查找已有的 SPEC 文件，确定下一个可用编号:
   ```
   SPEC-001, SPEC-002, SPEC-003, ...
   ```

3. **创建 Spec 文件**
   基于 `docs/specs/SPEC-TEMPLATE.md` 模板创建新的 spec:
   - 文件名: `docs/specs/SPEC-{NNN}-{简短标题}.md`
   - 填入用户提供的信息
   - 状态设为 `📝 草稿`
   - 填入日期和作者

4. **宪法合规检查**
   对照 `docs/specs/CONSTITUTION.md` 检查 spec 是否合规:
   - 是否包含测试策略？
   - 是否评估了性能影响？
   - 是否考虑了安全因素？
   - 是否识别了受影响的架构边界？

5. **输出 Spec 摘要 (中文)**
   向用户呈现 spec 的中文摘要，等待审批。

> [!IMPORTANT]
> Spec 创建后状态为「草稿」，必须经过用户审批才能进入实施阶段。
