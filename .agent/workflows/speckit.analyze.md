# /speckit.analyze

对现有 spec、实施计划或代码进行深度分析。

## 步骤

1. **确定分析目标**
   询问用户要分析什么:
   - `spec` — 分析某个 spec 的质量和完整性
   - `plan` — 分析实施计划的可行性
   - `code` — 分析已实施代码与 spec 的一致性
   - `all` — 全面分析项目状态

2. **Spec 分析 (当目标为 spec)**
   检查以下维度:
   - ✅ 完整性: 是否填写了所有必要字段
   - ✅ 清晰度: 目标和范围是否明确
   - ✅ 可测试性: 验收标准是否可量化
   - ✅ 宪法合规: 是否符合项目宪法的所有原则
   - ✅ 风险评估: 是否识别了关键风险

3. **计划分析 (当目标为 plan)**
   检查以下维度:
   - ✅ 可执行性: 步骤是否足够细化
   - ✅ 依赖管理: 是否正确标注了依赖关系
   - ✅ 测试覆盖: 测试计划是否充分
   - ✅ 回滚策略: 是否有可行的回滚方案
   - ✅ 影响范围: 是否超出了 spec 声明的范围

4. **代码分析 (当目标为 code)**
   - 对比 spec 和实际实现
   - 检查是否有偏离 spec 的地方
   - 检查测试覆盖情况
   - 检查性能和安全合规性

5. **生成分析报告**
   输出包含以下内容的报告:
   ```
   ## 分析报告: {目标}
   
   ### 评分
   | 维度 | 评分 (1-5) | 说明 |
   |------|-----------|------|
   
   ### 问题
   - 🔴 严重: ...
   - 🟡 警告: ...
   - 🔵 建议: ...
   
   ### 改进建议
   1. ...
   2. ...
   ```

6. **输出摘要 (中文)**
   完整的分析结果以中文呈现。

> [!NOTE]
> 此工作流不修改任何文件，仅产生分析报告。
