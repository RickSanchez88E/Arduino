# 设计: OS-001 - ArUco 视觉定位导航

## 做法概要

我们将在用作上位机的 PC 或者单板计算机上建立一个全新的视觉处理模块 `aruco_tracker.py`。
使用 `cv2.aruco` 库分析顶置摄像头采集的实时画面。为了消除镜头的畸变或者斜视角度带来的三维透视投影误差：
1. 我们预先规定场景中 4 个特定 ID 的 Tag 作为静态参考锚点（比如定义为真实物理场景 2米x2米 的四个绝对坐标 `(0,0)`, `(2,0)`, `(2,2)`, `(0,2)`），实时计算单应性变换矩阵（Homography）。
2. 将镜头里看到的小车顶置 Tag，提取其中心像素坐标，利用求出的矩阵，进行透视变换计算，映射到真实的米制世界坐标系 $(x, y)$ 当中。
3. 利用小车顶部 Tag 的 4 个角点之间的连线朝向，结合相机的旋转或者坐标系的变换，算出相对于全局坐标系的全局航向角 $\theta$。

然后可以通过回调函数、进程间通信或 Socket 线程来将这些值提供给已有的 Tkinter 地面控制站主程序进行状态监控或最终用于取代轮式里程计。

## 接口变更

- 视觉模块应当封装为一个类（如 `ArucoLocalization`），提供形如 `.start_camera()`, `.get_pose()` 等方法或传入一个接收 `(x, y, theta)` 的回调钩子。
- 此模块需无缝运行在单独的守护进程或线程中，不应阻塞主程序的 Tkinter 窗口循环及与小车的 TCP Socket 通信信道。

## 已知风险

- **风险 1**: Windows 使用有线 USB 摄像头往往存在缓存和延迟，如果帧率太低，获取的位置滞后会导致控制出现超调和震荡。
- **风险 2**: 照明不足、曝光过度或运动产生的动态模糊导致 ArUco tag 丢失（False Negative），不能给出连续读数。需要预留插值或者沿用旧读数防抖的逻辑。

## 绕不开的约束

- 无需复杂 3D 相机内参标定即可获得相当精度的平面定位，但必须依赖 4 个角作为基础平面标定。若角点被遮挡，此时无法正确计算或需要引入更高级的历史跟踪算法，为简单起见目前假设这 4 个锚点总是在视野内。（也可以放宽条件只用其中部分）。
