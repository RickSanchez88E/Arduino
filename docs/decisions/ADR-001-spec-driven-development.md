# ADR-001: 采用规范驱动开发流程

> 状态: ✅ 已采纳
> 日期: 2026-03-13
> 决策者: Bootstrap Agent (经用户授权)

## 背景

AG Workflow Builder 是一个新项目，需要建立可持续、可追溯的开发流程，特别是在使用 AI agent 进行辅助开发的场景下。

## 决策

采用规范驱动开发 (Spec-Driven Development) 流程:

```
spec → 审批 → 计划 → 任务分解 → 实施 → 验证 → 更新文档
```

使用 Antigravity 原生概念（Rules, Workflows, Skills）实现完整的 Spec Kit 工作流。

## 理由

1. **可追溯性**: 每个代码变更都能追溯到一个已批准的 spec
2. **质量保证**: 强制的审批流程减少了错误引入
3. **多 Agent 协作**: 明确的任务边界减少了并发冲突
4. **中文亲和**: 所有面向用户的产出以中文呈现

## 影响

- 开发速度: 初期学习曲线，但长期提升质量和效率
- 文档维护: 需要持续更新 project-context 和 spec 状态

## 替代方案

1. 直接开发无spec → 拒绝: 缺乏追溯性
2. 使用外部项目管理工具 → 拒绝: 增加工具切换成本
